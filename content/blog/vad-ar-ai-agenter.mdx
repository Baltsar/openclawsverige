AI-agenter är program som inte bara svarar på frågor — de **utför uppgifter**. Till skillnad från en vanlig chatbot som skickar tillbaka text, kan en agent använda verktyg, fatta beslut och köra flera steg i sekvens utan att du behöver styra varje moment.

En chatbot svarar. En agent agerar.

## Agenter vs chatbots

Skillnaden handlar om autonomi.

| | Chatbot | Agent |
|---|---|---|
| **Svarar med text** | ✓ | ✓ |
| **Gör saker** (API-anrop, beräkningar) | ✗ | ✓ |
| **Svar per tur** | Ett | Flera steg |
| **Planerar** | Nej | Ja |
| **Minne** | Begränsat | Kan spara kontext |
| **Verktyg** | Inga | Function calling, MCP, API:er |

En chatbot är som att skicka ett SMS och få svar. En agent är som att ge en kollega en uppgift och låta dem lösa den.

## Hur fungerar det tekniskt?

Tre koncept att förstå:

### Function calling (tool use)

Modellen får en lista med funktioner den *kan* anropa. Istället för att bara generera text, returnerar den ett strukturerat anrop — din kod kör funktionen och skickar resultatet tillbaka.

Exempel: du frågar "Vad är vädret i Stockholm?" — modellen anropar `get_weather("Stockholm")`, får tillbaka data, och formulerar ett svar baserat på resultatet.

Det stöds av OpenAI, Anthropic, Google och de flesta modell-API:er.

### MCP (Model Context Protocol)

MCP är en öppen standard (skapad av Anthropic, släppt som open source) för hur AI-modeller kopplas till externa verktyg och datakällor. Tänk USB-C för AI — ett standardiserat sätt att exponera verktyg så att olika klienter och modeller kan använda dem.

Istället för att varje integration är custom-byggd, definierar MCP ett gemensamt protokoll. En MCP-server kan exponera vad som helst: databasuppslag, API-anrop, filhantering, affärssystem.

### Multi-agent-system

Flera agenter som samarbetar. En agent researchar, en skriver kod, en granskar. De kommunicerar med varandra och delar upp arbetet.

Ramverk som CrewAI, AutoGen och LangGraph gör det enklare att orkestrera sådana system. Men det går att bygga från scratch också — det handlar om att ge varje agent en roll, verktyg och ett sätt att kommunicera.

## Ett enkelt exempel i Python

Här är ett minimalt exempel där en agent har tillgång till ett väderverktyg:

```python
from openai import OpenAI

client = OpenAI()

# Definiera verktyget
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Hämta väder för en stad",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "Stadens namn"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

# Skicka fråga med verktyg tillgängliga
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Vad är vädret i Stockholm?"}],
    tools=tools,
    tool_choice="auto"
)

# Modellen returnerar ett tool_call istället för bara text
# Din kod kör funktionen och skickar resultatet tillbaka
```

Raderna 3–26 definierar verktyget. Resten är ett vanligt API-anrop med `tools` och `tool_choice` tillagt. Modellen bestämmer själv om den behöver anropa verktyget.

Det här är function calling i sin enklaste form. En riktig agent lägger till: loopar (kör flera steg), minne (spara kontext mellan anrop), och felhantering (försök igen om något misslyckas).

## Varför det växer nu

Tre saker hände:

**Modellerna blev bättre på att använda verktyg.** GPT-4, Claude 3.5, Gemini — alla stöder function calling med hög tillförlitlighet. Det som var experimentellt 2023 är produktionsredo 2025.

**MCP standardiserade integrationerna.** Innan MCP var varje verktygsintegration custom. Nu finns ett gemensamt protokoll, och ekosystemet av färdiga MCP-servrar växer snabbt.

**Ramverken mognade.** CrewAI, AutoGen, LangGraph och andra gör det praktiskt att bygga multi-agent-system utan att skriva allt från scratch.

## Vad folk faktiskt bygger

Det här är inte science fiction. Folk bygger agenter som:

- Automatiserar bokföring och fakturering mot svenska affärssystem
- Övervakar och sammanfattar branschnyheter
- Hanterar kundärenden från inbox till lösning
- Genererar rapporter från databaser
- Kopplar ihop tjänster via MCP-servrar

Det gemensamma: **uppgifter som tidigare krävde en människa som klickade runt mellan system.** En agent kan göra samma sak, snabbare, 24/7.
